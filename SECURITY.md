# Security Policy

## Reporting a Vulnerability

If you find a security issue in any v2e-sh repository, please report it
privately. Do **not** open a public issue for security problems.

- Email: void0x2e@proton.me
- Or use GitHub's private vulnerability reporting ("Report a vulnerability"
  under the Security tab of the affected repo).

Please include:

- The repository and affected file(s) or component.
- A description of the issue and its potential impact.
- Steps to reproduce, if available.

Expect an initial response within a few days. Please allow reasonable time to
investigate and ship a fix before any public disclosure.

## Scope

These are homelab infrastructure repositories. Findings that involve committed
secrets, insecure defaults, or supply-chain risks (unverified downloads, image
pinning) are especially in scope.
